# ぐらびゅ3 プロジェクトコンセプト

## 概要

**ぐらびゅ3** は、Windows用画像ビューアソフトウェア「ぐらびゅ2」の後継として開発する軽量・高速な画像ビューアである。

ぐらびゅ2はC++ / Win32 API / Visual Studio 2019で開発されていたが、以下の課題があった：

- **開発環境の構築が面倒**: Visual Studioのインストールが必要
- **複雑化した機能**: 画像編集機能（フィルタ、レベル補正、モザイク等）が増え、ビューア本来の目的から外れた肥大化
- **自作フレームワーク(Themis)への依存**: メンテナンスコストが高い

ぐらびゅ3では、**必要な機能に絞ったシンプルなビューア**としてゼロから作り直す。

## 設計方針

### 1. シンプルさの追求

ぐらびゅ2にあった画像編集機能（フィルタ、レベル補正、ぼかし、シャープ、モザイク、色反転、グレースケール化、切り抜き、解像度変更等）は**全て削除**する。ぐらびゅ3は**画像を見ることに特化**する。

### 2. 高速な画像切り替え

先読み（プリフェッチ）機能を中核に据え、**瞬時の画像切り替え**を実現する。これはぐらびゅ2で最も評価された機能であり、ぐらびゅ3でも最優先で実装する。

### 3. モダンな開発環境

Visual Studioを不要とし、`rustup` + テキストエディタ（VSCode推奨）で開発できるようにする。

## 技術選定

### 言語: Rust

| 観点 | 評価 |
|------|------|
| パフォーマンス | C++同等。先読みバッファの精密なメモリ管理に所有権システムが最適 |
| 開発環境構築 | `rustup` 一発 + Visual Studio Build Tools（軽量、IDE不要） |
| スレッド安全性 | コンパイル時にデータ競合を検出。先読みスレッドの実装が安全 |
| FFI (Susieプラグイン) | `libloading` + `extern "system"` で64bit DLLの動的ロードが可能 |
| エコシステム | 画像処理(`image`, `turbojpeg`)、アーカイブ(`zip`, `unrar`, `sevenz-rust`)が充実 |

**検討した代替案:**

- **C# .NET 8+**: 開発は容易だが、GCのpauseが先読みバッファの大量メモリ管理に不利。WPFの本格開発はVisual Studio推奨になりがち。
- **C++ (MinGW/CMake)**: パフォーマンスは問題ないが、CMake等のビルド設定が煩雑でモダンとは言い難い。

### GUI: windows-rs (Win32 API) + Direct2D

- **windows-rs**: Microsoft公式のRust用Win32バインディング。型安全にWin32 API、COM、Direct2Dを呼び出せる。
- **Direct2D**: GPU加速による高速な画像描画。WIC (Windows Imaging Component) と連携して画像デコード→描画のパイプラインを構築。
- ぐらびゅ2のWin32ウィンドウ + GDI描画モデルを踏襲しつつ、描画エンジンをDirect2Dに置き換える。

### 主要ライブラリ

| 機能 | crate | 備考 |
|------|-------|------|
| Win32 API | `windows` (Microsoft公式) | ウィンドウ管理、Direct2D、WIC、Shell API |
| 画像デコード | `image` + `turbojpeg` | 汎用デコード + JPEG高速デコード |
| PNGメタデータ | `png` | コメント・メタデータ取得 |
| ZIP | `zip` | Unicode対応 |
| RAR | `unrar` | 静的リンク、外部DLL不要 |
| 7-Zip | `sevenz-rust` | Pure Rust、外部依存なし |
| DLLロード | `libloading` | Susieプラグイン用 |
| スレッド間通信 | `crossbeam-channel` | 先読みスレッドとのメッセージング |
| 設定ファイル | `serde` + `toml` | 設定のシリアライズ/デシリアライズ |
| エラーハンドリング | `anyhow` + `thiserror` | アプリ/ライブラリレベル |

### ビルド・開発フロー

```bash
# 環境構築（初回のみ）
# 1. rustup インストール（公式サイト）
# 2. Visual Studio Build Tools インストール（C++ビルドツールのみ）
# 3. VSCode + rust-analyzer 拡張

# 日常の開発
cargo build              # ビルド
cargo run                # 実行
cargo run -- image.jpg   # 画像ファイルを指定して実行
cargo test               # テスト
cargo clippy             # 静的解析
cargo fmt                # コードフォーマット

# リリースビルド
cargo build --release    # 最適化ビルド（LTO有効）
# → target/release/gv3.exe
```

## ぐらびゅ2からの変更方針

### 削除する機能

| 機能 | 理由 |
|------|------|
| 画像編集（フィルタ全般） | ビューアに不要。レベル補正、ぼかし、シャープ、モザイク、色反転、グレースケール化等 |
| 見開き表示 | シンプル化 |
| スライドショー | シンプル化 |
| 拡大鏡 (CMagnifierWindow) | シンプル化 |
| ネットワーク更新 | 配布方法を別途検討 |
| GDI/GPU描画モード切替 | Direct2D一本に統一 |
| 壁紙設定機能 | ビューアに不要 |
| 外部ビューア連携 | シンプル化 |
| 自作フレームワーク(Themis) | Rustエコシステムのcrateで代替 |

### 継承する機能

- ファイルリスト管理（ナビゲーション、シャッフル、ソート、ブックマーク）
- 先読みエンジン（ぐらびゅ2の `ThreadedImageLoader` + `PageLoaderBase` の設計を踏襲）
- マーク機能（一括ファイル操作）
- カスタマイズ可能なキーバインド
- アーカイブ対応（ZIP, RAR, 7z）
- Susieプラグイン対応（64bit）
- シェル統合（関連付け、右クリックメニュー、送る）

### 新規・改善

- **TOML設定ファイル**: ぐらびゅ2の独自形式(.cfg)からTOMLへ移行。人間が読み書きしやすい
- **GPU加速描画**: GDI → Direct2Dへの移行により、大画像の拡大縮小が高速化
- **GIF対応の標準化**: Susieプラグインなしでも対応
- **WebP対応**: `image` crateの標準対応
